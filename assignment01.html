Purpose

The purpose of Lab & Assignment 01 is to bring all students to the same baseline level of semantic HTML skill with minimal CSS and no JavaScript. Your work should clearly communicate the conceptual differences between Norman’s model, GOMS, and KLM.

Starter Project

You will receive a starter website repository (5 pages). Your task is to improve the HTML structure, clarity, and accessibility while keeping within the rules below.

  • index.html (Home)
  • cv.html (Conceptual CV / structured content page)
  • contact.html (Contact form)
  • analysis.html (Norman vs. GOMS comparison table)
  • norman2klm.html (Norman vs. GOMS vs. KLM + operators)

You may add content related to our Week 01–Week 04 discussion (e.g., Toggle Switch, Automatic Syringe, Who Paid Problem), but all work must remain within Lab 01 scope and the Norman/GOMS/KLM comparison theme.

Content Scope (Strict)

Use only content related to:

  • Norman’s Action Cycle (and gulfs of execution/evaluation)
  • GOMS (goals, operators, methods, selection rules)
  • KLM (operators and execution-time prediction)

Source restriction: Use selective and relevant content from Dix et al., Human–Computer Interaction (Ch. 9 and Ch. 12). You may add hand-designed UI sketches/layouts for interaction analysis, but do not introduce unrelated topics or external sources.

HTML Requirements (Tabular)

HTML Requirements for Lab & Assignment 01 (Semantic Structure, IA, and Accessibility Basics)
Requirement Area HTML Element(s) Rule (What to Do) Purpose (Why) Common Mistakes to Avoid
Semantic Structure header, nav, main,
section, article, footer 		Use semantic elements according to their meaning. Improves clarity, maintainability, and screen-reader navigation. Using only div; misusing nav.
Navigation Structure nav, ul, li, a Navigation must be structured as a list of links. Screen readers expect menus as lists of destinations. Missing li; inconsistent navigation.
Headings h1–h4 Use one h1 per page; do not skip levels. Creates a clear information hierarchy. Multiple h1; skipping levels.
Tables table, caption, thead, tbody, th, td Use tables only for comparison data. Improves accessibility and comprehension. Using tables for layout; missing headers.

Accessibility Requirements

  • Use aria-current="page" correctly
  • All form controls must have associated labels
  • Images must include appropriate alt text
  • Site must be usable via keyboard

CSS Rules (Strict: Minimal Only)

You may use CSS only for:

  • color
  • background-color
  • font-family
  • font-size
